# Miners REST API

Учебный backend-проект на Go: симуляция предприятия "Угольная шахта" с управлением через HTTP API.

## Что делает проект
- Запускает предприятие с пассивной добычей угля.
- Позволяет нанимать шахтёров разных классов (в том числе пачкой).
- Позволяет покупать оборудование за накопленный уголь.
- Возвращает текущий статус предприятия и финальное время игры при завершении.

## Стек
- Go
- Gorilla Mux

## Запуск
```bash
go run ./cmd/app
```

Сервер запускается на `:8080`.

## Игровая модель (текущее состояние)
- Пассивный доход: `+1` уголь каждые `2` секунды.
- Классы шахтёров:
  - `weak`: цена `5`, энергия `30`, добыча `1`, интервал `3с`, без прогрессии.
  - `normal`: цена `50`, энергия `45`, добыча `3`, интервал `2с`, без прогрессии.
  - `strong`: цена `450`, энергия `60`, добыча `10`, интервал `1с`, прогресс `+3` за цикл.
- Оборудование:
  - `pickaxe`: `3000`
  - `ventilation`: `15000`
  - `wagon`: `50000`

## Уведомления по балансу
В статусе предприятия есть `notifications` — это milestones по балансу без спама.  
Сейчас шаг уведомлений: `250` угля, хранится до `8` последних записей.

## API

### 1) Шахтёры

`GET /miners/prices`  
Вернуть параметры классов шахтёров.

`POST /miners/hire?class={class}&count={count}`  
Нанять шахтёров выбранного класса.
- `class`: `weak | normal | strong`
- `count`: целое число `>= 1`

`GET /miners/active`  
Список активных шахтёров.

`GET /miners/active?class={class}`  
Фильтр активных шахтёров по классу.

### 2) Оборудование

`GET /equipment/prices`  
Список цен на оборудование.

`POST /equipment/{type}/buy`  
Купить оборудование.
- `type`: `pickaxe | ventilation | wagon`

`GET /equipment`  
Текущее состояние: какое оборудование куплено.

### 3) Предприятие

`GET /enterprise/status`  
Текущий снимок предприятия:
- баланс,
- активные шахтёры,
- статистика найма,
- оборудование,
- уведомления.

`POST /enterprise/shutdown`  
Завершить игру и вернуть длительность работы предприятия.

### 4) Приложение

`PUT /app/close`  
Принудительно завершить приложение (`os.Exit(0)` после короткой задержки).

## Примеры запросов

```bash
curl http://localhost:8080/miners/prices
```

```bash
curl -X POST "http://localhost:8080/miners/hire?class=weak&count=3"
```

```bash
curl http://localhost:8080/enterprise/status
```

```bash
curl -X POST http://localhost:8080/equipment/pickaxe/buy
```

```bash
curl -X POST http://localhost:8080/enterprise/shutdown
```

## Структура проекта
- `cmd/app` — точка входа.
- `internal` — доменные типы и конфигурация.
- `logic` — бизнес-логика предприятия и жизненный цикл горутин.
- `myHttp` — transport слой (роуты, DTO, хендлеры).
